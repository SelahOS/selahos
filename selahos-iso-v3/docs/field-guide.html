2. Creating a Bootable USB

You will need a USB drive of at least 8GB. All data on the drive will be erased. Download the ISO from selahos.io/downloads.

On macOS

Open Terminal. Use diskutil list to find your USB drive, then replace diskX below:

diskutil list
diskutil unmountDisk /dev/diskX
sudo dd if=selahos-1.0.1-beta-x86_64.iso of=/dev/rdiskX bs=4m status=progress
diskutil eject /dev/diskX

On Linux

lsblk
sudo dd if=selahos-1.0.1-beta-x86_64.iso of=/dev/sdX bs=4M status=progress oflag=sync

On Windows

Use Rufus (rufus.ie) — select the ISO, choose GPT partition scheme, and write in DD Image mode.

Note: Verify the SHA256 checksum matches the one posted at selahos.io/downloads before flashing.

3. Boot & Installation

Booting from USB

  • On Apple Intel Macs: Hold the Option key at startup, select the EFI Boot drive
  • On PC: Press F12, F2, or Del at startup to access boot menu
  • Select the SelahOS USB drive to boot the live environment

Running the Installer

  • The installer launches automatically on first boot
  • Select your target disk — WARNING: this will erase the selected disk
  • Enter your username, password, and hostname
  • Click Install — the process takes 10-20 minutes
  • When complete, the system will prompt you to reboot

If the installer does not launch

sudo python3 /usr/local/bin/selahos-installer

If the installer shows an error at the end

⚠ Important — Read This: In Beta 1.0.1, the installer may display an unmount error or fail message at the very end of installation. This does not mean the installation failed. The system files have already been written successfully to disk before this step runs.
✓ How to confirm your install succeeded: Reboot the machine. If SelahOS boots and you reach the login screen, the installation completed successfully. Proceed to the First Boot Checklist below to apply remaining components.

To check the install log for details:

cat /tmp/selahos-install.log | tail -50

4. First Boot Checklist

Follow these steps in order after your first successful boot into SelahOS. Each step builds on the previous one.

  1. Connect to the internet
    Open the network icon in the taskbar and connect to WiFi or plug in Ethernet. Internet is required for all remaining steps.
  2. Open Konsole (Terminal)
    Right-click the desktop → Open Terminal, or find Konsole in the app launcher.
  3. Run the Post-Install Fix script
    This fixes mirrors, kdesu, multilib, theme, and launchers in one shot. 
    curl -fsSL https://raw.githubusercontent.com/SelahOS/selahos/main/selah-postinstall-fix.sh | sudo bash
  4. Log out and back in
    Click the application menu → Leave → Log Out. Log back in with your credentials. This activates all permission and theme changes.
  5. Install SelahBridge
    Open Konsole again and run: 
    curl -fsSL https://raw.githubusercontent.com/SelahOS/selahos/main/selahbridge-install.sh | sudo bash
  6. Log out and back in again
    Required to activate SelahBridge audio group permissions.
  7. Run the system audit (optional but recommended)
    Confirms everything is working correctly. 
    curl -fsSL https://raw.githubusercontent.com/SelahOS/selahos/main/selah-audit.sh | sudo bash
  8. Launch your first Windows application
    Open SelahBridge from the app launcher and enjoy.
Note: All scripts are pulled directly from github.com/SelahOS/selahos. You must have an active internet connection to run them.

5. Post-Install Setup

The post-install fix script is the most important thing to run after a fresh install. It corrects all known Beta 1.0 issues automatically.

Run from GitHub

# From github.com/SelahOS/selahos
curl -fsSL https://raw.githubusercontent.com/SelahOS/selahos/main/selah-postinstall-fix.sh | sudo bash

What it fixes

  • Installs kdesu — required for application launchers on Wayland
  • Enables multilib repository — required for SelahBridge
  • Updates package mirrors for reliable downloads
  • Synchronizes package databases
  • Applies the Selah color scheme to your user profile
  • Applies hardware stability fixes for Apple Intel machines
  • Fixes all desktop launcher files (pkexec → kdesu)

After the script completes, log out and back in to activate all changes.

Full System Audit

Report only — shows what needs fixing:

# From github.com/SelahOS/selahos
curl -fsSL https://raw.githubusercontent.com/SelahOS/selahos/main/selah-audit.sh | sudo bash

Auto-fix mode — repairs all detected issues:

# From github.com/SelahOS/selahos
curl -fsSL https://raw.githubusercontent.com/SelahOS/selahos/main/selah-audit.sh | sudo bash -s -- --fix

6. SelahBridge Setup

SelahBridge is SelahOS's proprietary Windows application compatibility layer. It enables Windows creative software — including professional DAWs and music production tools — to run natively on SelahOS.

Installing via GUI

  • Open the Application Launcher
  • Search for SelahBridge Setup
  • Click Install — the process takes 5-15 minutes
  • Log out and back in when prompted

Installing via Console (Recommended for Beta)

Run directly from the official GitHub repository:

# From github.com/SelahOS/selahos
curl -fsSL https://raw.githubusercontent.com/SelahOS/selahos/main/selahbridge-install.sh | sudo bash
Note: Make sure you have run selah-postinstall-fix.sh first. If mirrors are not configured, SelahBridge installation will fail.

Confirmed Working Applications

  • InMusic Software Center — installs via SelahBridge, authorize with Falkon browser ✓
  • Additional DAWs and creative tools — in active testing
Note: After SelahBridge installation, always log out and back in before launching Windows applications. This activates audio group permissions.

10. Troubleshooting

Most issues — run this first

# From github.com/SelahOS/selahos — fixes most known Beta 1.0.1 issues
curl -fsSL https://raw.githubusercontent.com/SelahOS/selahos/main/selah-postinstall-fix.sh | sudo bash

No WiFi after install

sudo pacman -S broadcom-wl-dkms
sudo modprobe wl

Black screen after boot

sudo systemctl restart sddm

Audio not working

systemctl --user restart pipewire wireplumber
groups $USER   # should include: audio realtime

Or run the full audit and auto-fix from GitHub:

# From github.com/SelahOS/selahos
curl -fsSL https://raw.githubusercontent.com/SelahOS/selahos/main/selah-audit.sh | sudo bash -s -- --fix

SelahBridge fails to install

# Step 1 — fix mirrors first (from github.com/SelahOS/selahos)
curl -fsSL https://raw.githubusercontent.com/SelahOS/selahos/main/selah-postinstall-fix.sh | sudo bash

# Step 2 — retry SelahBridge (from github.com/SelahOS/selahos)
curl -fsSL https://raw.githubusercontent.com/SelahOS/selahos/main/selahbridge-install.sh | sudo bash

Fix mirrors manually

sudo tee /etc/pacman.d/mirrorlist << 'EOF'
Server = https://geo.mirror.pkgbuild.com/$repo/os/$arch
Server = https://mirror.rackspace.com/archlinux/$repo/os/$arch
EOF
sudo pacman -Sy

Check install log

cat /tmp/selahos-install.log | tail -50
